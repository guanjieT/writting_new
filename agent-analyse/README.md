## 一句话概括

`novel_agent_clean` 是一个以项目为中心的小说生成工作流系统。用户先创建小说项目，再沿着“需求 -> 世界观 -> 角色 -> 总纲 -> 卷纲 -> 章节计划 -> 正文 -> 审查/记忆”的链路逐步生成结构化产物。后端负责工作流约束、上下文组装、LLM 调用和产物落盘；前端负责表单、项目选择、任务轮询和产物展示。

## 关键原则

- 后端路由不直接写业务逻辑，只把请求转换为任务。
- Agent 不直接读写文件，只负责上下文、Prompt 和 LLM 调用。
- 项目、产物、任务和审计都可回看，便于追踪生成链路。
- 工作流依赖和作用域统一定义，避免前后端各写一套隐性规则。
- 规划类产物以结构化摘要为主体，正文 `content` 只作为可读摘要和审查材料。
